# Security policy

This application runs as root inside its container so it can access Unraid-owned Docker template XML files, the two narrowly scoped Docker Manager icon-cache directories, the configured Compose Manager projects folder, and the Docker socket. Saving an icon only changes metadata and caches. The explicit sync action uses the Docker API to recreate only selected containers; for Compose Manager containers it also atomically updates the selected service's icon label in `docker-compose.override.yml`. Docker socket and host write mounts are privileged by nature. It is intended for a trusted local network only. Do not expose it directly to the internet.

Production startup requires an `ADMIN_TOKEN` of at least 24 characters. The Web UI exchanges it only for an in-memory, `HttpOnly`, `SameSite=Strict` 12-hour session; management APIs require that session, and every write additionally requires a same-origin CSRF token. Set `TRUSTED_NETWORKS` to the smallest CIDR range that contains your Unraid host and browsers (for example `192.168.2.0/24`). The default allows RFC1918 LAN ranges and loopback to avoid locking out standard Unraid installations. Only hash-addressed icon/wallpaper file URLs stay unauthenticated so Unraid can render them; they reveal no inventory or management data.

The supplied deployment profiles drop all Linux capabilities, use a read-only root filesystem with a small no-exec `/tmp`, and set CPU, memory and PID limits. Those controls reduce blast radius but **do not make a mounted Docker socket safe**: Docker's API itself remains host-equivalent. Run no reverse proxy with public exposure in front of this service. Prefer a dedicated management VLAN or firewall rule if it must be reachable by more than one LAN segment.

Do not replace the documented mounts with broad `/boot`, `/var/lib/docker`, `/usr/local/emhttp`, or `/mnt/user` mounts. Mount only Compose Manager's configured `PROJECTS_FOLDER` at `/unraid/compose-projects`. Sync preserves the selected container's binds, named volumes, ports, environment, restart policy and networks; it does not delete volumes or recreate unrelated Compose services. If recreation fails, the service attempts to restore the original container and override, but users should still maintain normal appdata backups.

External icon and wallpaper URLs are downloaded by the service before they enter a gallery. The importer accepts only HTTP(S), limits DNS/connection/total time, redirects and bytes, rejects credentials, remote SVG and non-image responses, and blocks loopback, private, link-local, metadata and reserved network addresses. Public custom ports remain supported because many self-hosted public icon repositories use them; only validated raster image bytes are retained. Local/private image sources should be uploaded through the Web UI instead. Do not weaken these checks on a service that has Docker and Unraid host access.

Report vulnerabilities privately to the repository owner rather than opening a public issue. Do not attach diagnostics containing container labels, mount paths, WebUI URLs, or credentials.
